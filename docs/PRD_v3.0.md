# DocuMind 产品需求文档 v3.0

## 一、产品设计理念

### 1.1 核心原则

**文件即一切**

- 用户上传的**文件**是系统唯一的核心实体
- AI分析结果、关键词、摘要、分类、版本历史都是**文件的属性**
- 所有文件**自动索引**到知识库
- 上传入口**只在工作台**

### 1.2 核心概念图

```
┌─────────────────────────────────────────────────────┐
│                     文件（File）                      │
│  ┌───────────────────────────────────────────────┐  │
│  │ 基础属性                                        │  │
│  │ - 文件名、类型、大小                            │  │
│  │ - 物理存储路径（MinIO）                         │  │
│  │ - 所属用户                                      │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │ AI分析属性                                      │  │
│  │ - 摘要（summary）                               │  │
│  │ - 关键词（keywords）                            │  │
│  │ - 分类（category）                              │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │ 版本属性                                        │  │
│  │ - 当前版本号                                    │  │
│  │ - 版本历史（历史版本链）                        │  │
│  │   └── v1: 物理文件路径 + AI结果                 │  │
│  │   └── v2: 物理文件路径 + AI结果                 │  │
│  │   └── v3: 物理文件路径 + AI结果                 │  │
│  └───────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────┐  │
│  │ 知识库属性                                      │  │
│  │ - 索引状态（indexed=true）                      │  │
│  │ - 向量ID                                        │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 1.3 用户真实需求

```
用户在工作台上传文件 → 系统自动AI分析 → 显示摘要/关键词/分类
                       → 文件自动进入知识库 → AI问答可检索
                       → 文件有版本历史 → 可追溯、可下载历史版本
```

**用户不需要理解复杂概念**，用户只关心：
1. 在哪里上传文件？→ **工作台**
2. AI帮我分析了什么？→ **摘要、关键词、分类**
3. 这个文件有哪些版本？→ **版本历史（每个版本可下载）**
4. 能搜索到这个文件吗？→ **所有文件自动索引**

## 二、数据模型设计

### 2.1 单表设计：file_info（唯一核心表）

```sql
-- 文件表（唯一核心实体）
CREATE TABLE file_info (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    
    -- ===== 基础属性 =====
    file_uuid       VARCHAR(36) UNIQUE NOT NULL,    -- 文件唯一标识（用于物理存储）
    file_name       VARCHAR(255) NOT NULL,           -- 显示文件名（用户可修改）
    original_name   VARCHAR(255) NOT NULL,           -- 原始上传文件名
    file_type       VARCHAR(50),                     -- MIME类型
    file_size       BIGINT DEFAULT 0,                -- 文件大小(bytes)
    storage_path    VARCHAR(500) NOT NULL,           -- 当前版本物理存储路径(MinIO)
    
    -- ===== AI分析属性 =====
    summary         TEXT,                            -- AI摘要
    keywords        JSON,                            -- AI关键词(JSON数组)
    category        VARCHAR(100) DEFAULT '其他',     -- AI分类
    
    -- ===== 版本属性 =====
    version         INT DEFAULT 1,                   -- 当前版本号
    version_history JSON,                            -- 版本历史(JSON数组)
    /*
    version_history结构示例:
    [
        {
            "version": 1,
            "file_uuid": "uuid-v1",
            "original_name": "论文A.pdf",
            "storage_path": "minio/path/v1",
            "file_size": 102400,
            "summary": "v1摘要...",
            "keywords": ["AI", "深度学习"],
            "category": "论文",
            "change_summary": "初始版本",
            "create_time": "2026-05-17T10:00:00"
        },
        {
            "version": 2,
            "file_uuid": "uuid-v2",
            ...
        }
    ]
    */
    
    -- ===== 知识库属性 =====
    indexed         BOOLEAN DEFAULT TRUE,            -- 是否已索引（默认true）
    vector_id       VARCHAR(100),                    -- 向量数据库ID
    index_time      DATETIME,                        -- 索引时间
    
    -- ===== 处理状态 =====
    process_status  VARCHAR(20) DEFAULT 'COMPLETED', -- 处理状态
    process_time    DATETIME,                        -- 处理完成时间
    
    -- ===== 用户归属 =====
    user_id         BIGINT NOT NULL,
    status          VARCHAR(20) DEFAULT 'ACTIVE',    -- ACTIVE/DELETED
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_user_id (user_id),
    INDEX idx_category (category),
    INDEX idx_status (status),
    INDEX idx_create_time (create_time)
);
```

### 2.2 版本历史数据结构

**version_history字段存储所有历史版本，每个版本包含：**

```json
{
    "version": 1,
    "file_uuid": "abc123-v1",           // 该版本的物理文件UUID
    "original_name": "论文A.pdf",        // 该版本的原始文件名
    "storage_path": "minio/users/5/abc123-v1.pdf", // 物理存储路径
    "file_size": 102400,
    "summary": "本文研究深度学习...",    // 该版本的AI摘要
    "keywords": ["深度学习", "AI", "神经网络"],
    "category": "论文",
    "change_summary": "初始版本",
    "create_time": "2026-05-17T10:00:00Z"
}
```

**设计优势：**
1. **历史版本可追溯**：version_history包含所有历史
2. **物理文件可下载**：每个版本都有storage_path
3. **AI结果可查看**：每个版本都有独立的summary/keywords
4. **单表查询**：无需关联多表

### 2.3 与v2.0对比

| 维度 | v2.0（冗余） | v3.0（统一） |
|-----|-------------|-------------|
| 表数量 | 4个(file+doc+doc_version+ai_task) | 1个(file_info) |
| 核心实体 | file + doc分离 | file统一 |
| AI结果存储 | doc_info + doc_version重复 | file_info单一存储 |
| 版本历史 | doc_version独立表 | file_info.version_history字段 |
| 知识库索引 | 概念模糊 | file_info.indexed=true |
| 查询复杂度 | 多表JOIN | 单表查询 |

## 三、核心功能设计

### 3.1 文件上传（只在工作台）

```
入口：工作台页面唯一上传入口
       ┌─────────────────────────────────────────────────────┐
       │  📤 拖拽文件上传                                      │
       │  上传后自动AI分析 + 自动进入知识库                     │
       └─────────────────────────────────────────────────────┘

流程：
1. 用户在工作台拖拽上传文件
2. 系统检测相似度：
   - 无相似文件 → 创建新文件(version=1)
   - 有相似文件 → 弹出选择框：新建文件 OR 更新版本
3. 存储物理文件到MinIO
4. 自动AI分析：摘要、关键词、分类
5. 自动索引到知识库（indexed=true）
6. WebSocket推送完成通知

关键点：
- 所有文件上传都在工作台
- 文件中心只能查看，不能上传
- 上传后自动索引，无需手动操作
```

### 3.2 版本管理

```
场景：用户上传相似文件"论文A_v2.pdf"

检测相似度 → 相似度>=80% → 弹出选择框
┌─────────────────────────────────────────────────────┐
│ ⚠️ 检测到相似文件                                    │
│ 相似文件: 论文A.pdf (v1)                             │
│ 相似度: 92%                                          │
│                                                      │
│ 请选择处理方式：                                      │
│ ○ 创建新文件（独立文件）                             │
│ ● 更新版本（推荐）  版本号将变为 v2                   │
│   [输入变更说明：添加实验数据________________]        │
│                                                      │
│              [取消]  [确认上传]                       │
└─────────────────────────────────────────────────────┘

选择"更新版本"后：
1. 当前版本(v1)信息保存到version_history
2. 新版本(v2)成为当前版本
3. 更新file_info：
   - version: 1 → 2
   - storage_path: 新MinIO路径
   - summary/keywords: 新AI结果
   - version_history: 添加v1历史记录
```

- 点击"下载此版本" → 下载该版本的物理文件

```

### 3.4 知识库索引（自动）

```
索引策略：
- 所有文件上传后自动索引（indexed=true）
- 向量存储：以file_uuid为key
- 内容来源：解析后的文本内容

索引流程：
1. 上传文件 → 存MinIO → 解析文本
2. 调用AI向量服务 → 生成向量
3. 存入向量数据库 → 更新file_info.vector_id
4. file_info.indexed = true

索引查询：
- AI问答 → 向量检索 → 返回file_uuid
- 前端显示 → 文件名 + 摘要片段
- 点击 → 打开文件详情页

版本更新索引：
- 新版本上传 → 新内容重新索引
- vector_id更新为新向量ID
- 旧版本索引保留（可查询历史版本）
```

### 3.5 分类机制（AI自动）

```
分类类别：
├── 论文           - 学术论文、研究报告
├── 技术文档       - API文档、技术规范、代码文档
├── 分析报告       - 数据分析、市场报告、可行性报告
├── 会议记录       - 会议纪要、讨论记录
├── 试卷           - 考试题目、练习题、作业
├── 合同文档       - 合同、协议、法律文件
├── 通知公告       - 公告、通知、说明
└── 其他           - 无法归类

分类流程：
1. 解析文件内容
2. AI分析标题+内容 → 返回分类
3. 存储到file_info.category
4. 用户可手动修改分类
```

## 五、API设计

### 5.1 文件API

```
工作台上传（唯一入口）：
POST   /api/files/upload              # 上传文件（自动检测相似度）
       Request: { file, userId }
       Response: { 
           file_uuid, 
           similarity_detected: true/false,
           similar_file: { file_uuid, file_name, version } // 如果有相似
       }

POST   /api/files/upload-new          # 确认创建新文件
       Request: { file_uuid (temp), file_name, userId }

POST   /api/files/upload-version      # 确认更新版本
       Request: { 
           file_uuid (temp), 
           existing_file_uuid,  // 要更新的文件
           change_summary, 
           userId 
       }

文件查询：
GET    /api/files                     # 获取文件列表
       Query: { userId, category?, status?, keyword? }
       Response: { files: [{ file_uuid, file_name, version, category, summary_preview, indexed }] }

GET    /api/files/{file_uuid}         # 获取文件详情（含完整version_history）
       Response: { 
           file_uuid, file_name, version, category, 
           summary, keywords, indexed,
           version_history: [{ version, storage_path, summary, keywords, ... }]
       }

GET    /api/files/{file_uuid}/download        # 下载当前版本
GET    /api/files/{file_uuid}/download/{version} # 下载指定版本

文件修改：
PUT    /api/files/{file_uuid}/category        # 修改分类
       Request: { category }
       
DELETE /api/files/{file_uuid}                 # 删除文件（软删除）
```

### 5.2 AI分析API

```
POST   /api/ai/summarize              # 生成摘要
       Request: { content }
       Response: { summary }

POST   /api/ai/keywords               # 提取关键词
       Request: { content }
       Response: { keywords: ["词1", "词2"] }

POST   /api/ai/classify               # 分类
       Request: { title, content }
       Response: { category: "论文" }

POST   /api/ai/qa                     # 知识库问答
       Request: { question, userId }
       Response: { 
           answer, 
           sources: [{ file_uuid, file_name, version, summary_preview }]
       }
```

## 七、验收标准

### 7.1 功能验收

| 功能 | 验收标准 |
|-----|---------|
| 文件上传 | 只能在工作台上传，上传后自动AI分析 |
| 相似检测 | 相似度>=80%弹出选择框，可选择新建/更新版本 |
| 版本历史 | 每个版本可下载物理文件、查看AI结果 |
| 知识库索引 | 所有文件自动索引，indexed=true |
| AI问答 | 问答返回来源文件，点击打开详情页 |
| 分类机制 | AI自动分类，用户可手动修改 |

### 7.2 数据验收

| 检查项 | 验收标准 |
|-------|---------|
| 单表设计 | 只有file_info一个核心表 |
| 无冗余 | summary/keywords/version只存一次 |
| 版本完整 | version_history包含所有历史版本信息 |
| 索引完整 | 所有文件indexed=true |

### 7.3 性能验收

| 指标 | 目标 |
|-----|------|
| 上传响应 | <1s（不含AI处理） |
| AI处理时间 | <30s（PDF<10页） |
| 版本历史查询 | <100ms（单表查询） |
| 知识库检索 | <2s |

## 八、总结

### 8.1 核心设计要点

| 要点 | 说明 |
|-----|------|
| 单一实体 | 文件是唯一核心概念，无"文档"概念 |
| 版本内嵌 | version_history作为JSON字段存储历史 |
| 历史可下载 | 每个版本都有storage_path，可下载物理文件 |
| 上传集中 | 只在工作台上传，其他页面只查看 |
| 全量索引 | 所有文件自动索引到知识库 |
| 单表查询 | 无多表JOIN，查询简单高效 |

### 8.2 用户视角

```
用户操作：
1. 在工作台上传文件 → 自动分析 → 看到结果
2. 上传相似文件 → 选择更新版本 → 版本号递增
3. 在文件中心查看所有文件 → 点击看详情
4. 点击版本历史 → 下载任意历史版本
5. AI问答 → 看到来源文件 → 点击查看详情
6. 分类自动 → 可手动修改

用户不需要知道：
- 什么是"文档"
- 什么是"知识库索引"
- 什么是"向量"
- 后端有多少表
```

### 8.3 与v2.0核心差异

```
v2.0:
- file + doc分离（4个表）
- 上传入口分散（工作台+文件中心）
- 版本历史独立表
- 知识库索引概念模糊
- AI结果存储冗余

v3.0:
- file统一（1个表）
- 上传入口唯一（工作台）
- 版本历史内嵌JSON
- 所有文件自动索引
- AI结果存储唯一
```

```

│ │───────────────────────────────────────────────────││
│ │ v2 | 昨天 | 论文A_v2.pdf                           ││
│ │     摘要: 添加实验数据...                          ││
│ │     [📥 下载] [🔍 查看分析] [🔄 切换为当前版本]     ││
│ │───────────────────────────────────────────────────││
│ │ v1 | 3天前 | 论文A.pdf                             ││
│ │     摘要: 初始版本...                              ││
│ │     [📥 下载] [🔍 查看分析] [🔄 切换为当前版本]     ││
│ └───────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

切换版本流程：
1. 用户点击"切换为当前版本"
2. 弹出确认框：
   ┌─────────────────────────────────────────────────────┐
   │ ⚠️ 确认切换版本                                      │
   │                                                      │
   │ 将 v2 设为当前版本后：                               │
   │ - 当前版本号变为 v2                                  │
   │ - 当前显示的摘要/关键词变为 v2 的内容                │
   │ - v3 不会删除，仍保留在版本历史中                    │
   │                                                      │
   │ [取消]  [确认切换]                                   │
   └─────────────────────────────────────────────────────┘
   
3. 系统更新file_info：
   - version: 3 → 2
   - storage_path: v2的MinIO路径
   - summary/keywords: v2的AI结果
   - version_history: 不变（仍包含v1,v2,v3）

4. 向量索引更新：
   - vector_id: 更新为v2内容的向量
   - AI问答将基于v2内容回答

设计要点：
- 切换版本不删除任何历史记录
- 切换后重新生成向量索引
- 当前版本标记可变，但版本历史永久保留
```

### 9.2 文件删除与恢复

```
软删除机制：
┌─────────────────────────────────────────────────────┐
│ file_info.status 字段                                │
│ - ACTIVE: 正常状态                                   │
│ - DELETED: 已删除（软删除）                          │
│                                                      │
│ 删除时不物理删除文件，只标记status=DELETED            │

删除流程：

   ┌─────────────────────────────────────────────────────┐

   │                                                      │
   │ [取消]  [确认删除]                                   │

   - file_info.status = 'DELETED'
   - file_info.delete_time = now()
   - 调用vector-service移除向量
   - 向量数据库标记删除（软删除）

恢复机制：
┌─────────────────────────────────────────────────────┐
│ 🗑 已删除文件                                          │
│ ┌───────────────────────────────────────────────────┐│

│ │───────────────────────────────────────────────────││

│ └───────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

恢复流程：
1. 用户点击"恢复"
2. file_info.status = 'ACTIVE'
3. file_info.delete_time = null
4. 重新索引到向量数据库

永久删除（30天后或用户手动）：
1. 物理删除MinIO文件
2. 物理删除向量数据
3. 物理删除file_info记录

设计要点：
- 软删除保留30天，给用户恢复窗口
- 删除后立即从知识库移除（向量软删除）
- 恢复后重新索引
```

    password        VARCHAR(100),                -- 可选密码（加密存储）
    download_limit  INT DEFAULT -1,              -- -1=不限
    download_count  INT DEFAULT 0,               -- 已下载次数
    expire_time     DATETIME,
    status          VARCHAR(20) DEFAULT 'ACTIVE',
    create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_share_code (share_code)
);

公开访问流程：
1. 用户访问 https://documind.com/share/abc123
2. 如果有密码 → 输入密码验证
3. 显示文件预览页面（无需登录）
4. 如有下载权限 → 可下载
5. 每次下载记录 download_count++

设计要点：
- 分享不改变文件所有权
- 分享不影响知识库索引（被分享者不会自动拥有）
- 过期分享自动失效
```

### 9.7 知识库向量管理

```
向量生命周期：
┌─────────────────────────────────────────────────────┐
│                                                      │
│  文件上传 → 生成向量 → 存入向量库 → 持续可用         │
│                                                      │
│  文件删除 → 标记删除 → 向量不可检索 → 保留元数据     │
│                                                      │
│  文件恢复 → 取消删除标记 → 向量恢复可用              │
│                                                      │
│  永久删除 → 物理删除向量 → 完全移除                  │
│                                                      │
│  版本更新 → 新向量覆盖旧向量（旧版本向量可选保留）   │
│                                                      │
│  版本切换 → 生成对应版本向量                         │
│                                                      │
└─────────────────────────────────────────────────────┘

向量存储结构（Milvus/PGVector）：
┌─────────────────────────────────────────────────────┐
│ 向量表 schema                                        │
│                                                      │
│ 字段：                                               │
│ - id              VARCHAR(100)  // 向量ID           │
│ - embedding       FLOAT[1536]   // 向量数据         │
│ - file_uuid       VARCHAR(36)   // 文件UUID         │
│ - user_id         BIGINT        // 用户ID（隔离）   │
│ - version         INT           // 版本号           │
│ - content_hash    VARCHAR(64)   // 内容hash         │
│ - summary         TEXT          // 摘要（用于展示） │
│ - file_name       VARCHAR(255)  // 文件名           │
│ - category        VARCHAR(100)  // 分类             │
│ - is_deleted      BOOLEAN       // 删除标记         │
│ - create_time     TIMESTAMP     // 创建时间         │
│                                                      │
│ 主键: file_uuid + version                           │
│ 索引: user_id, is_deleted                          │
└─────────────────────────────────────────────────────┘

向量索引策略：
┌─────────────────────────────────────────────────────┐
│ 新文件上传：                                          │
│ - 解析内容 → 生成向量                                │
│ - 存入向量库，is_deleted=false                       │
│ - 更新file_info.vector_id                           │
│                                                      │
│ 版本更新：                                            │
│ - 新内容生成新向量                                   │
│ - 旧向量标记is_deleted=true（保留历史）             │
│ - 新向量关联新version                                │
│                                                      │
│ 版本切换：                                            │
│ - 切换到v2 → 查找v2向量                              │
│ - 若v2向量存在 → 取消其删除标记                      │
│ - 若v2向量不存在 → 重新生成                          │
│                                                      │
│ 文件删除：                                            │
│ - 向量is_deleted=true                                │
│ - 向量不再参与检索                                   │
│                                                      │
│ 文件恢复：                                            │
│ - 向量is_deleted=false                               │
│ - 向量恢复可检索                                     │
│                                                      │
│ 永久删除：                                            │
│ - 物理删除向量记录                                   │
└─────────────────────────────────────────────────────┘

向量检索查询：
┌─────────────────────────────────────────────────────┐
│ 用户提问 → 向量检索                                  │
│                                                      │
│ 查询条件：                                            │
│ - user_id = 当前用户ID                               │
│ - is_deleted = false                                 │
│ - 返回top_k=5最相似结果                              │
│                                                      │
│ 返回结果：                                            │
│ [                                                    │
│     {                                                │
│         "file_uuid": "uuid_abc",                    │
│         "file_name": "论文A.pdf",                   │
│         "version": 2,                               │
│         "summary": "本文研究...",                   │
│         "similarity": 0.85                          │
│     },                                               │
│     ...                                              │
│ ]                                                    │
└─────────────────────────────────────────────────────┘

历史版本向量保留策略：
┌─────────────────────────────────────────────────────┐
│ 配置选项：                                            │
│                                                      │
│ retain_history_vectors = true/false                 │
│                                                      │
│ true：                                               │
│ - 每个版本的向量都保留                               │
│ - AI问答时可选择查询特定版本                         │
│ - 占用更多存储空间                                   │
│                                                      │
│ false：                                              │
│ - 只保留当前版本向量                                 │
│ - 版本更新后旧向量物理删除                           │
│ - 节省存储空间                                       │
│                                                      │
│ 默认：true（保留历史向量）                           │
└─────────────────────────────────────────────────────┘
```
